# PLAN-2-2: исправление небезопасного хранения управляющих артефактов демона

Источник: `ISSUE.md` (пункт 2), `docs/rfcs/0002.md` (Фаза 1, пункт 2).

## Допущения

- Речь идет об артефактах управления демоном: PID, stop-signal file, runtime log и связанных marker/status файлах.
- Исправление должно быть минимальным и не менять архитектуру демона.
- Проверки выполняются через цели `Makefile`.

## Цель

Убрать хранение daemon-control артефактов из `/tmp` и перейти на приватный runtime-каталог пользователя с корректными правами доступа.

## Требуемое поведение

1. Базовый путь:
   - при наличии `XDG_RUNTIME_DIR`: `${XDG_RUNTIME_DIR}/ssh-key-sync/<sid>/`;
   - fallback: `${HOME}/.local/run/ssh-key-sync/<sid>/`.
2. Права:
   - каталог(и): `0700`;
   - файлы: `0600`.
3. Создание файлов:
   - использовать безопасные флаги создания (без небезопасного follow/truncate поведения).
4. Все операции daemon start/stop/status/log работают с новым путем и не обращаются к `/tmp`.

## Границы (in scope / out of scope)

### In scope

- Вынести вычисление runtime-пути в единый helper.
- Перевести все места записи/чтения control-файлов на новый helper.
- Добавить тесты path-selection и прав доступа.

### Out of scope

- Ротация логов.
- Изменение формата control-файлов.
- Изменение сетевого протокола или trust-модели.

## План работ

## Этап 1. Инвентаризация и единый runtime-path helper

Задачи:

1. Найти все обращения к `/tmp` и все места работы с daemon-control артефактами.
2. Ввести один источник истины для пути runtime-каталога (с учетом `sid` и fallback-логики).
3. Нормализовать создание runtime-каталога с `0700`.

Проверка:

1. Unit-тесты выбора пути:
   - `XDG_RUNTIME_DIR` задан;
   - `XDG_RUNTIME_DIR` отсутствует, используется fallback в `~/.local/run/...`.

## Этап 2. Миграция PID/stop/log/status файлов

Задачи:

1. Перенести создание PID/stop/log/status файлов в новый runtime-каталог.
2. В местах создания файлов явно выставлять `0600`.
3. Использовать безопасные флаги открытия/создания файлов во всех control-flow точках.
4. Удалить прямые обращения к `/tmp` из daemon-control кода.

Проверка:

1. Unit/интеграционные тесты на start/stop/status с новым путем.
2. Тесты прав доступа:
   - каталог `0700`;
   - файлы `0600`.

## Этап 3. Совместимость CLI и документация

Задачи:

1. Проверить, что CLI-команды управления демоном используют тот же helper пути.
2. Обновить `docs/man.md` (разделы daemon/control files), указав новый путь и fallback.
3. Зафиксировать поведение при отсутствии прав на создание runtime-каталога (явная ошибка).

Проверка:

1. Команды daemon start/stop/status в тестовом окружении работают без `/tmp`.
2. Документация отражает фактический runtime-путь и права.

## Критерии завершения

- В кодовой базе нет использования `/tmp` для daemon-control артефактов.
- Runtime-каталог создается только как приватный (`0700`), control-файлы — `0600`.
- Поведение start/stop/status/log сохраняется, путь хранения артефактов предсказуем и документирован.
- Проверки проходят через:
  - `make fmt`
  - `make lint`
  - `make test`
  - `make build`

## Риски и контроль

- Различия поведения прав/umask на разных ОС: минимизировать явной установкой permissions и тестами.
- Возможные старые артефакты в `/tmp`: не читать их как источник истины после миграции.
- Ошибки прав в `XDG_RUNTIME_DIR`: fallback в `~/.local/run/...` только при отсутствии переменной, а не при silent-ignore ошибок создания.
